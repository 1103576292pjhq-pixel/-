# NPU 背景教程 01：从零理解这道题在做什么

## 1. NPU 是什么
NPU 可以理解成“为了神经网络和矩阵乘法而专门设计的计算器”。  
和 CPU 相比，NPU 不擅长复杂分支和通用程序控制，但非常擅长把大量重复的乘加运算并行做掉。  
和 GPU 相比，NPU 往往更强调：

- 更固定、更窄的计算数据通路
- 更适合矩阵乘和张量运算的本地存储结构
- 更高的能效比

## 2. 这道题为什么像一个 NPU 小核心
赛题要求你做的是一个 `32x16` 的块浮点计算阵列。  
它不是整个大模型推理芯片，但它已经具备了 NPU 的典型特征：

- 有阵列化并行计算单元
- 以矩阵乘为主要目标
- 对低精度数值格式做专门优化
- 强调吞吐、面积和功耗，而不是通用性

## 3. 什么是 MAC
MAC 是 `Multiply-Accumulate`，中文可以理解成“乘加”。  
例如：

```text
acc = acc + a * b
```

神经网络里的大部分核心计算，最后都可以拆成大量 MAC。

## 4. 什么是矩阵乘
矩阵乘本质上是很多个点积：

```text
Y[i][j] = sum_k A[i][k] * B[k][j]
```

所以硬件里通常会先实现“点积引擎”，再把很多点积引擎并排组成阵列。

## 5. 什么是 PE、阵列、数据流
- `PE`：Processing Element，处理单元
- 阵列：很多 PE 按行列组织起来，同时工作
- 数据流：输入、权重、输出部分和在阵列里的流动方式

本项目里可以把每个 `llmt_col` 理解成一列点积处理单元，16 列组成一个小阵列。

## 6. 为什么需要低精度
如果一直用 `FP32`：

- 乘法器和加法器更大
- 存储和搬运的数据更多
- 功耗更高

所以大模型硬件常常会用更低精度格式，提高吞吐并降低能耗。

## 7. 什么是块浮点
块浮点的意思是：一组数据共享同一个缩放因子。  
这样做的好处是：

- 单个元素可以更短
- 还能保持比较大的动态范围

这道题里的 `MXFP8` 就属于块浮点思路：

- block 内 32 个元素用 `E4M3`
- 整个 block 共享 1 个 `E8M0 scale`

## 8. 这套设计里有哪些关键模块
你可以先把整个项目想成 6 层：

1. `E4M3/E8M0` 格式解析
2. 单元素乘法
3. `dot32` 归约
4. `FP32` 累加
5. `32x16` 阵列组织
6. 验证、综合、报告和说明文档

## 9. 为什么矩阵要切 tile，尾 tile 怎么办
因为本项目的阵列一次只能并行算 `16` 列输出，所以矩阵乘不会“一次把所有列都做完”，而是要按列方向切成一个个 tile。

例如 `N = 20` 时：

- 第一个 tile 处理前 `16` 列
- 第二个 tile 只剩下后 `4` 列是真实数据

这时硬件或 testbench 通常会把剩下的 `12` 个空 lane 用零输入补齐。这样做有两个好处：

- 阵列接口保持固定宽度，不需要为了最后一小块再换一套端口协议
- 可以明确验证“没用到的 lane 不应该残留旧结果”

所以一个成熟的矩阵级回归，不能只测 `16`、`32` 这种整齐尺寸，也要测 `20` 这种会出现尾 tile 的情况。
